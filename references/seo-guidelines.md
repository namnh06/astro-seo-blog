# SEO Guidelines for Blog Posts

Complete SEO best practices following Google Search Central guidelines.

---

## Google Search Central Best Practices

### People-First Content (Core Principle)
- Write for humans first, search engines second
- Provide genuine value and answer real questions
- Demonstrate first-hand expertise and knowledge
- Focus on helping users accomplish their goals
- Content should be worth sharing or bookmarking

### E-E-A-T Framework

| Pillar | How to Demonstrate |
|--------|-------------------|
| **Experience** | Share real scenarios, client stories, local observations |
| **Expertise** | Include technical details, professional tips, industry knowledge |
| **Authoritativeness** | Cite reputable sources when making claims |
| **Trustworthiness** | Be accurate, transparent, honest about expectations |

### Content Quality Standards

**DO:**
- Provide original, unique insights based on real experience
- Include specific, actionable steps users can follow
- Use descriptive, helpful titles that summarize content accurately
- Write comprehensive content that fully addresses the topic
- Keep content current and accurate
- Make content easy to read with clear structure

**DON'T:**
- Create content solely to rank in search engines
- Use keyword stuffing or unnatural repetition
- Generate thin, low-value content
- Copy content from other sources
- Create misleading or clickbait titles
- Write generic content without unique value

## Content Structure

### Opening Section (First 150 words)

**Must Include:**
- Engaging, conversational opening that relates to reader's problem
- Clear statement of what the article will solve
- Local relevance (mention location when appropriate)
- Natural keyword integration
- Personal touch showing expertise

**Avoid:**
- Generic, boring openings ("In this article, we will discuss...")
- Overly formal corporate language
- Starting with obvious statements
- Walls of text without personality

### Main Content Structure

```
H1: [Main Title - only one per page]

[Engaging Opening - 2-3 paragraphs]

H2: [First Main Section]
[Content with actionable information]

H2: [Second Main Section]
[Content with actionable information]

H2: [Third Main Section]
[Content with actionable information]

H2: [Professional Help / CTA Section]
[Link to relevant service with benefits list]

H2: Related Articles
[3-4 internal blog post links]
```

### Section Guidelines
- Each H2 section: 150-300 words
- Use bullet points for lists and steps
- Use **bold** for emphasis
- Include practical, step-by-step instructions
- Short paragraphs (2-4 sentences max)

## SEO Technical Requirements

### Title Tags
- 50-60 characters maximum
- Include primary keyword naturally
- Add location when relevant
- Format: `[Topic] | [Secondary Info] | [Business Name]`

### Meta Descriptions
- 150-160 characters maximum
- Compelling summary that encourages clicks
- Include primary keyword naturally
- Mention location for local SEO

### URL Structure
- Use kebab-case (lowercase with hyphens)
- Keep URLs short and descriptive
- Include primary keyword
- Example: `/blog/first-time-home-buyer-guide`

### Heading Hierarchy
- Only one H1 per page (the main title)
- Use H2 for main sections
- Use H3 for subsections
- Never skip levels (H1 → H2 → H3, not H1 → H3)

## Internal Linking Strategy

### Link Frequency
| Link Type | Minimum per Post |
|-----------|-----------------|
| Total internal links | 6-8 |
| Service page links | 2-3 |
| Related blog posts | 3-4 |
| Homepage/Contact | 1-2 |

### Anchor Text Guidelines

**Good Anchor Text:**
- Descriptive and specific
- Natural within sentence flow
- Includes keywords when relevant
- Examples: "website development service", "contact us today"

**Bad Anchor Text:**
- "Click here"
- "Read more"
- "This link"
- Over-optimized keyword stuffing

## Writing Style & Tone

### Voice & Personality
- **Conversational** - Write like talking to a friend
- **Helpful** - Focus on solving problems, not selling
- **Local** - Mention location when relevant
- **Expert but Approachable** - Show knowledge without being condescending
- **Friendly** - Use contractions, personal pronouns (you, we, I)

### Sentence Structure
- Vary sentence length
- Active voice preferred
- Short paragraphs (2-4 sentences max)
- Use transitions: However, Meanwhile, Additionally, First, Next, Finally

### Word Choice
- Simple, clear language (8th-grade reading level)
- Avoid jargon unless necessary (explain technical terms)
- Choose specific words over generic ones

### Local SEO Integration
- Mention location 3-5 times naturally throughout content
- Use local references (neighborhoods, landmarks, seasonal factors)
- Reference local events or conditions when relevant

## Quality Checklist

### Content Quality
- [ ] Unique topic not covered in existing posts
- [ ] 1,000-2,000 words (appropriate for topic)
- [ ] Opening hooks reader within first 3 sentences
- [ ] Content provides genuine value and actionable steps
- [ ] Tone is conversational and helpful
- [ ] Location mentioned 3-5 times naturally
- [ ] No spelling or grammar errors

### Structure & Formatting
- [ ] Only one H1 (the main title)
- [ ] H2 sections are well-organized
- [ ] H3 used for subsections only
- [ ] Paragraphs are 2-4 sentences max
- [ ] Bullet points used for lists
- [ ] Bold used for emphasis appropriately

### Internal Linking
- [ ] 6-8 total internal links minimum
- [ ] 2-3 links to service pages
- [ ] 3-4 links to related blog posts
- [ ] 1-2 links to contact/homepage
- [ ] All anchor text is descriptive

### SEO Technical
- [ ] Title is 50-60 characters
- [ ] Meta description is 150-160 characters
- [ ] URL slug matches filename (kebab-case)
- [ ] Date is set to today's date
- [ ] Frontmatter matches config format

### Calls-to-Action
- [ ] Professional help section included
- [ ] Service benefits listed with bullet points
- [ ] Strong CTA linking to contact page
- [ ] Local emphasis in CTA section
